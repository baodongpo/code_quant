## 1. 本次迭代目标

**主题：从"能看"到"看得懂"——面向初学者的界面体验跃升**

| 迭代 | 目标 |
|------|------|
| 迭代3 | 能看、能分析——技术指标可视化，辅助判断后市方向 |
| **迭代3.5（本迭代）** | **看得懂、用得顺——初学者友好的界面优化，降低信息理解门槛** |
| 迭代4 | 基本面数据、备用数据源、归档清理 |

**核心问题诊断**：

当前界面（迭代3）存在两类核心痛点：

| 痛点类型 | 具体表现 | 初学者影响 |
|---------|---------|----------|
| **信息难以决策** | 各指标信号分散在不同副图右上角，无综合结论 | 看完 MACD、RSI、KDJ 各说各的，不知道该怎么操作 |
| **布局过于紧凑** | 副图各 140px，六个区域堆叠，信息密度高 | 视觉疲劳，注意力分散，难以聚焦关键信息 |

**本迭代不改动范围**：
- 后端 API 接口（无需变更）
- 指标计算逻辑（IndicatorEngine 不改动）
- 数据采集逻辑（main.py / futu_wrap 不涉及）

## 3. 功能需求详述

### F1. 综合操作建议看板（综合信号横幅）

**优先级**：P0

**初学者视角的价值**：
初学者面对 MACD 金叉、RSI 超买、KDJ 中性三条信息时，往往无法形成决策，甚至产生矛盾感。综合横幅将多指标汇聚成一个通俗结论，帮助用户"先看结论、再看依据"，极大降低决策门槛。

> **产品立场**：综合建议是辅助阅读工具，不是智能投顾，每次展示均带有显著免责提示（常驻，不可关闭）。

#### 3.1.1 综合信号算法规范

基于现有后端已返回的信号字段，前端按以下规则聚合：

**信号得分映射表**：

| 指标 | 条件 | 得分 |
|------|------|------|
| MACD | DIF > DEA（多头持续）| +2 |
| MACD | DIF 刚上穿 DEA（金叉，最近3根K线内）| +3 |
| MACD | DIF < DEA（空头持续）| -2 |
| MACD | DIF 刚下穿 DEA（死叉，最近3根K线内）| -3 |
| RSI | 30 ≤ RSI ≤ 50 （超卖回升中性偏多）| +1 |
| RSI | 50 < RSI ≤ 70（中性偏多）| +2 |
| RSI | RSI > 70（超买）| -1 |
| RSI | RSI < 30（超卖）| +1 |
| RSI | 20 ≤ RSI < 30（深度超卖，强反弹信号）| +2 |
| KDJ | K 上穿 D（金叉）且 K < 30 | +2 |
| KDJ | K 上穿 D（金叉）且 30 ≤ K < 80 | +1 |
| KDJ | K 下穿 D（死叉）且 K > 70 | -2 |
| KDJ | K 下穿 D（死叉）且 30 ≤ K ≤ 70 | -1 |
| BOLL | 价格 < 下轨（超卖突破）| +2 |
| BOLL | 价格 > 上轨（超买突破）| -2 |
| BOLL | 价格在轨道内且上方 1/3 区间 | +1 |
| BOLL | 价格在轨道内且下方 1/3 区间 | -1 |
| MA | MA5 > MA10 > MA20（多头排列）| +2 |
| MA | MA5 < MA10 < MA20（空头排列）| -2 |

**综合评分 → 结论映射**（红涨绿跌配色体系，见§10）：

| 总分 | 综合结论 | 显示颜色 | 展示文案 |
|-----|---------|---------|---------|
| ≥ +5 | 积极做多信号 | 🔴 红（#f85149）| **多项指标共振向上，技术面偏强** |
| +2 ～ +4 | 偏多 | 🔴 浅红（#ff7b72）| **技术面多头信号为主，可关注买点** |
| -1 ～ +1 | 中性观望 | ⚖️ 灰 | **多空信号分歧，建议观望等待方向** |
| -2 ～ -4 | 偏空 | 🟢 浅绿（#3fb950）| **技术面空头信号为主，注意风险** |
| ≤ -5 | 积极做空信号 | 🟢 绿（#2ea043）| **多项指标共振向下，技术面偏弱** |

> **计分说明**：评分仅基于当日最新K线状态，纯前端计算，不写入数据库。

#### 3.1.2 UI 原型描述

**位置**：插入在顶部导航栏与主图之间（页面顶部横幅），高度 80px，全宽卡片式设计。

```
┌────────────────────────────────────────────────────────────────────┐
│  📊 技术面综合参考      贵州茅台 SH.600519   日K · 2026-03-18      │
│                                                                    │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │  ● MACD 金叉 +3   ● RSI 中性偏多 +2   ● KDJ 金叉 +2        │  │
│  │  ● MA 多头排列 +2                                            │  │
│  │                              综合得分: +9                    │  │
│  │  🔴  多项指标共振向上，技术面偏强（偏多方向）                  │  │
│  └──────────────────────────────────────────────────────────────┘  │
│  ⚠️ 以上为技术指标机械判断，仅供参考，不构成投资建议               │
└────────────────────────────────────────────────────────────────────┘
```

**各子模块说明**：
- **贡献因子 chips**：每条指标得分用浅色胶囊（chip）显示；买入信号（正分）= 红色系；卖出信号（负分）= 绿色系；中性（零分）= 灰色
- **综合得分**：数字 + 进度条（-12 到 +12 区间，0 居中，红色=偏多/绿色=偏空填充）
- **结论文案**：大字体（16px）加粗，颜色与结论映射表一致
- **免责声明**：小字（11px）灰色（`#bfbfbf`），**常驻显示，不可关闭**

**交互**：
- 点击看板区域展开/收起"指标说明面板"（见 F4 操作引导）
- 切换股票/周期后自动重新计算并刷新
- 悬停各因子 chip 时显示 Tooltip 详细说明该指标当前数值

### F2. 布局宽松化改造

**优先级**：P0

**初学者视角的价值**：
当副图紧密堆叠时，用户眼睛需要频繁在不同区域间切换，容易遗漏信号。宽松的间距和更高的面板使得每条曲线的走势清晰可辨，减少阅读疲劳。

#### 3.2.1 副图高度与间距调整

| 区域 | 当前高度 | 优化后高度 | 调整说明 |
|------|---------|----------|---------|
| 主图（K线+成交量）| 440px（K线54% + 成交量18%）| **500px** | 整体放大，K线区域更宽裕 |
| 综合信号横幅 | 无 | 80px | 新增 |
| MACD 面板 | 140px | **200px** | 统一放大，柱状图/线型更清晰 |
| RSI 面板 | 140px | **200px** | 统一放大 |
| KDJ 面板 | 140px | **200px** | 统一放大 |

> ⚠️ **高度 200px 为原型评审（2026-03-18）老板确认值，以此为准。** MACD/RSI/KDJ 三个副图高度统一为 200px，历史曾有 180px/160px/160px 的笔误记录，均以本行为准，QA 测试基准以 200px 为准。
| 各面板间距 | 0px（紧密贴合）| 8px | 增加视觉分隔 |
| 面板顶部 Padding | 4px | 12px | 标题区域更宽松 |

> **注意**：主图高度 500px 含 K 线区 + 成交量区两个子区域，不含侧边说明栏（侧边栏为固定 200px 宽度并排排列）。

**主图内部调整**：
- K线子图与成交量子图间距从 0 调整为 6px
- 成交量柱不再与 K 线混在同一 Grid，独立分区，比例调整为 K线70% + 成交量30%

#### 3.2.2 副图折叠功能

**需求描述**：用户可以自主选择显示哪些副图，折叠不关注的指标面板，聚焦核心信息。

**交互规范**：
- 每个副图面板右上角增加 **折叠按钮** `[ ∧ ]`（展开状态）/ `[ ∨ ]`（折叠状态）
- 折叠后面板高度收缩为 32px（仅显示面板标题 + 当前信号标签 + 展开按钮）
- **默认展开状态**：MACD（展开）、RSI（展开）、KDJ（展开）
- **状态持久化**：折叠状态**可选**保存到 `localStorage`（key: `quant_panel_collapse_state`），刷新页面后恢复用户的折叠偏好
- 折叠动画：200ms CSS transition，平滑收缩

**折叠后面板样式**：
```
┌────────────────────────────────────────────────────────────────────┐
│  RSI（相对强弱）          RSI14: 72.3   🟢 超买·注意回调    [ ∨ ]  │
└────────────────────────────────────────────────────────────────────┘
```

#### 3.2.3 宽屏自适应布局

**响应式断点规范**（桌面浏览器优先，与迭代3约束一致）：

| 视口宽度 | 布局行为 |
|---------|---------|
| < 1200px | 当前布局保持（上下堆叠），图表宽度100% |
| 1200px ～ 1600px | 综合看板与主图并排（看板固定宽320px，主图自适应剩余宽度）|
| > 1600px | 启用双栏布局：左栏主图+看板（65%），右栏副图纵向排列（35%）|

> **注意**：移动端不做要求（与迭代3保持一致），仅考虑桌面浏览器宽屏适配。

### F3. 主图买卖标记点增强

**优先级**：P0

**初学者视角的价值**：
当前信号仅通过副图右上角的小胶囊文字体现，初学者必须同时看多个副图才能找到买点。将关键信号直接标注在 K 线主图上，用户一眼就能看到"这根K线是个买入信号"。

#### 3.3.1 主图标记点规范（原型评审确认版）

在 K 线主图上叠加**圆形标记**（circle），仅标注 MACD 金叉/死叉两类核心事件：

| 事件 | 标记形状 | symbolSize | 位置 | 颜色 | 标记文字 | 字号 |
|------|---------|-----------|------|------|---------|------|
| MACD 金叉（DIF 上穿 DEA）| ● 圆形 | 12 | K线最低价下方 | #f85149（红）| 「买▲」| 17px 加粗 |
| MACD 死叉（DIF 下穿 DEA）| ● 圆形 | 12 | K线最高价上方 | #2ea043（绿）| 「卖▼」| 17px 加粗 |

> **设计原则**：
> - symbolSize=12（较小圆形），避免遮挡K线蜡烛本体
> - 标记文字「买▲」/「卖▼」清晰传达方向感（▲=向上/▼=向下）
> - 配色遵循红涨绿跌规则：买入标记=红圈，卖出标记=绿圈

**Tooltip 展示内容**（悬停标记点时）：
```
┌─────────────────────────────┐
│  📅 2026-01-15               │
│  MACD 金叉信号（买入参考）    │
│  DIF: 2.35  DEA: 2.28       │
│  ⚠️ 技术指标参考，非投资建议  │
└─────────────────────────────┘
```

**标记点显示控制**：
- 顶部导航栏增加"标记点开关"切换按钮（默认开启）
- 标记点密度保护：同一根K线上最多显示1个标记（优先显示第一个触发事件）

#### 3.3.2 信号标签视觉增强

**当前问题**：右上角胶囊文字 12px，视觉权重弱，初学者容易忽略。

**优化方案**：

| 元素 | 当前 | 优化后 |
|------|------|--------|
| 字体大小 | 12px | 13px（面板标签）|
| 最小宽度 | 无限制 | 100px（确保文案完整显示）|
| 圆角 | 4px | 6px |
| 内边距 | 4px 8px | 5px 10px |
| 显示位置 | 副图绝对定位右上角 | 副图标题栏右侧（文档流内，不遮挡图表）|
| 状态为"中性"时 | ⚖️灰色胶囊 | 同色但透明度降低 0.6（减少视觉干扰）|

**信号文案通俗化改版**（配色遵循红涨绿跌，见§10）：

| 原文案 | 新文案 | 颜色 |
|--------|--------|------|
| 🟢 金叉·多头信号 | 🔴 MACD 金叉，上升动能增强 | 红色系 |
| 🔴 死叉·空头信号 | 🟢 MACD 死叉，下行压力增大 | 绿色系 |
| 🔴 超买区间·注意回调 | 🟢 RSI 超买（72），短期或有回调 | 绿色系 |
| 🟢 超卖区间·关注反弹 | 🔴 RSI 超卖（25），关注反弹机会 | 红色系 |
| ⚖️ 中性区间 | ⚖️ RSI 中性（52），暂无明显信号 | 灰色 |
| 🟢 金叉·超卖买入 | 🔴 KDJ 金叉（低位），超卖反弹信号 | 红色系 |
| 🔴 死叉·超买卖出 | 🟢 KDJ 死叉（高位），超买回调信号 | 绿色系 |

### F4. 初学者操作引导

**优先级**：P1

**初学者视角的价值**：
初次打开界面的用户不了解指标含义，容易因看不懂而直接放弃。适时的提示和说明帮助用户建立"指标→含义→参考意义"的认知链路，提升留存和信任度。

#### 3.4.1 侧边说明栏（每个图表固定 200px）

**【原型评审确认】** 侧边说明不采用全局抽屉（Drawer），改为**每个图表右侧紧贴的固定 200px 说明栏**，与图表并排排列，始终可见。

**位置与尺寸**：
- 主图右侧：固定宽 200px 说明栏
- MACD/RSI/KDJ 副图各自右侧：固定宽 200px 说明栏
- 说明栏高度与对应图表一致
- 图表区域宽度 = 总容器宽度 - 200px

**说明栏内容规范**（每个副图各三块内容）：

```
┌──────────────────────────────┐
│  [当前指标值 + 信号标签]       │  ← 第一块：实时数值
│  RSI14: 72.3                 │
│  🟢 超买，注意回调             │
│  ─────────────────────────── │
│  [HTML 图例]                  │  ← 第二块：手动HTML图例
│  ─ RSI14  ████ (蓝线)        │
│  — 超买线70 ████ (红虚线)     │
│  — 超卖线30 ████ (绿虚线)     │
│  ─────────────────────────── │
│  [3条通俗解读]                 │  ← 第三块：说明文案
│  • RSI 衡量近期涨跌强弱        │
│  • >70 为超买，易回调          │
│  • <30 为超卖，可关注反弹      │
└──────────────────────────────┘
```

**图例规范**（不使用 ECharts 内置 legend）：
- 所有图表**禁用** ECharts `legend` 配置项
- 改用侧边栏内 **手动 HTML 图例**，用 `<div>` 色块 + 文字标注各线条颜色和含义
- 色块尺寸：12px × 12px 实心方块或 24px × 4px 线段色块，颜色与图表线条严格一致

#### 3.4.2 空状态与加载状态优化

| 状态 | 当前表现 | 优化后 |
|------|---------|--------|
| 数据加载中 | 空白 / 无提示 | 骨架屏（Skeleton）+ "数据加载中..." |
| 暂无数据 | 图表空白 | 友好提示卡片："该股票暂无足够数据（至少需要60个交易日）" |
| 接口请求失败 | 控制台报错 | 提示卡片 + 重试按钮 |

#### 3.4.3 底部信息条重组（减少信息过载）

**当前问题**：底部信息条将收盘价/PE/PB 与各指标信号标签堆在同一行，视觉密度高，初学者不知道看哪里。

**优化方案**：将底部信息条拆分为两层：

```
┌────────────────────────────────────────────────────────────────────┐
│  第一层（基础行情，始终可见）                                        │
│  收盘价: 1,830.00  涨跌: +1.2% ▲  PE: 28.5  PB: 9.2              │
├────────────────────────────────────────────────────────────────────┤
│  第二层（指标快览，可折叠，默认展开）              [ 折叠 ]          │
│  RSI: 72.3 🟢超买   MACD: 🔴金叉   KDJ: ⚖️中性   BOLL: ⚖️轨道内  │
└────────────────────────────────────────────────────────────────────┘
```

**设计原则**：
- 第一层始终可见，保证最核心行情数据不被遮挡
- 第二层默认展开，用户可折叠以获得更大图表空间
- 两层间加分割线，视觉层次分明

### F5. 副图标记点规范

**优先级**：P0

#### 3.5.1 MACD/KDJ 副图标记点（原型评审确认版）

### F7. 跨图时间轴联动（原型评审确认版）

**优先级**：P0

**需求描述**：所有图表（主图 + MACD + RSI + KDJ）共享同一时间轴，鼠标悬停时十字线在所有图上同步显示。

**技术方案**（不使用 `echarts.connect()`）：

> **原因**：`echarts.connect()` 在成交量区悬停时存在联动失效的已知问题，改用手动事件联动方案。

**实现规范**：
1. 监听**主图**的 `updateAxisPointer` 事件
2. 获取当前 `dataIndex`（时间索引）
3. 向 MACD/RSI/KDJ 三个副图实例各 `dispatchAction({ type: 'showTip', seriesIndex: 0, dataIndex })`
4. 监听主图 `mouseout` 事件 → 向所有副图 `dispatchAction({ type: 'hideTip' })`，清除副图浮窗

**技术要点**：
- 各图表实例通过 React `useRef` 持有
- 事件绑定在图表 `onEvents` 回调中完成
- 联动仅由主图驱动，副图不反向触发（避免事件循环）
