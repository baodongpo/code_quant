## 1. 测试环境

| 项目 | 说明 |
|------|------|
| 前端启动 | `cd web && npm install && npm run dev`（http://localhost:5173） |
| 后端启动 | `./env_quant/bin/uvicorn api.main:app --reload --port 8000` |
| 浏览器 | Chrome 最新版（主），Safari 最新版（次），Edge 最新版（次） |
| 分辨率 | 1440×900（主），1920×1080（宽屏测试），1280×800（小视口测试） |
| 数据要求 | watchlist.json 中至少3支不同市场股票；至少1支有效数据≥60根K线 |
| LocalStorage | 测试前清空 `quant_panel_collapse_state`（避免脏数据干扰） |

---

## 2. 配色约定（全局基准）

> 本迭代遵循 A股「红涨绿跌」原则，与原型 prototype_ui.html 保持一致：
>
> | 语义 | 颜色系 | 原型色值 |
> |------|--------|---------|
> | 买入信号 / 多头 / 金叉 / 超卖反弹 | **红色系** | `#f85149`（边框） / `#ff7b72`（文字） / `#3a1a1a`（背景） |
> | 卖出信号 / 空头 / 死叉 / 超买回调 | **绿色系** | `#2ea043`（边框） / `#3fb950`（文字） / `#1a3a2a`（背景） |
> | 中性 | 灰色系 | `#484f58`（边框） / `#8b949e`（文字） / `#1c2128`（背景） |
>
> **所有用例中"颜色正确"均以上表为基准进行验收。**

### 3.1 F1 — 综合信号横幅（SignalSummaryBar）

---

#### TC-F1-01：综合信号横幅基本渲染
- **优先级**：P0
- **前置条件**：页面加载完成，有效数据≥20根K线
- **操作步骤**：
  1. 打开 http://localhost:5173
  2. 选择任意有效股票，加载完成
- **预期结果**：
  - 综合信号横幅出现在顶部导航栏与主图之间
  - 横幅高度约 80px，全宽卡片样式，带圆角与边框
  - 显示以下元素：股票代码/名称、周期、综合结论文案（≥16px加粗）、免责声明文字

---

#### TC-F1-02：综合评分算法正确性验证
- **优先级**：P0
- **前置条件**：选择已知信号状态的3支不同股票（尽量覆盖多头/空头/中性）
- **操作步骤**：
  1. 打开浏览器开发者工具，在 Console 中手动获取当日末根K线的 MACD/RSI/KDJ/BOLL/MA 指标值
  2. 按 PRD §3.1.1 评分规则手动计算期望总分
  3. 对比界面上显示的综合得分
- **预期结果**：
  - 界面总分与手动计算结果一致（允许 ±0 误差，纯整数加减）
  - 各因子胶囊得分与评分规则映射表逐一对应

---

#### TC-F1-03：综合结论文案与颜色区间映射
- **优先级**：P0
- **前置条件**：需覆盖5个评分区间（需切换多支股票或等待不同数据状态）
- **操作步骤**：
  逐一验证每个分区间对应的文案与颜色：

  | 总分区间 | 预期文案 | 预期颜色 |
  |---------|---------|---------|
  | ≥ +5 | 多项指标共振向上，技术面偏强 | 🔴 红色系 |
  | +2 ～ +4 | 技术面多头信号为主，可关注买点 | 🔴 浅红系 |
  | -1 ～ +1 | 多空信号分歧，建议观望等待方向 | ⚖️ 灰色系 |
  | -2 ～ -4 | 技术面空头信号为主，注意风险 | 🟢 浅绿系 |
  | ≤ -5 | 多项指标共振向下，技术面偏弱 | 🟢 绿色系 |

- **预期结果**：文案和颜色与上表完全一致，无颠倒

---
