# AI 量化辅助决策系统 - 数据源子系统需求文档

**版本**: v1.0
**日期**: 2026-03-03
**范围**: 仅数据采集，绝对禁止任何自动交易逻辑

---

## 1. 背景与目标

从零搭建针对 A股 / 港股 / 美股的量化数据采集系统，为后续量化算法提供干净、完整、可复权的历史与实时K线数据。

**核心约束**：本系统仅负责数据采集与存储，严禁包含任何自动下单、报价、交易相关逻辑。

---

## 2. 数据范围

### 2.1 支持市场

| 市场 | 代码前缀 | 日历基准 | 币种 |
|------|---------|---------|------|
| A股（沪）| SH. | SH | CNY |
| A股（深）| SZ. | SH | CNY |
| 港股 | HK. | HK | HKD |
| 美股 | US. | US | USD |

### 2.2 数据品类

- **K线粒度**：日K（1D）、周K（1W）、月K（1M）
- **K线字段**：开盘价、最高价、最低价、收盘价、成交量（股）、成交额、换手率、市盈率（TTM，仅日K）、前收盘价
- **复权数据**：存储原始未复权价格 + 独立复权因子表，算法层动态转换前复权
- **交易日历**：各市场交易日，用于空洞检测基准

### 2.3 资产类型

支持普通股票（stock）和 ETF。

## 3. 功能需求

### 3.1 Watchlist 管理

- 用户通过编辑 `watchlist.json` 维护关注股票列表
- 每次启动时自动检测变化，支持三种场景：
  - **新增**：DB 中不存在 → 触发全量历史拉取（从 2000-01-01 起）
  - **重新激活**（`is_active: false → true`）→ 触发全量空洞检测与补全
  - **停用**（`is_active: true → false`）→ 取消实时订阅、暂停历史拉取，保留已有数据
- 无需重启服务，重新运行 `main.py` 即可生效

### 3.2 历史K线采集

- 数据源：富途 OpenD `get_history_kline` 接口
- 采集模式：
  - **全量**：新增股票，从 `DEFAULT_HISTORY_START` 起全量拉取
  - **增量**：已有数据，从上次成功同步日期续拉
  - **空洞修复**：检测并补全历史数据缺口
- 请求限频：双约束令牌桶（最小间隔 0.5s + 30s 窗口最多 25 次）
- 分页处理：单次最多 1000 条，超出自动翻页

### 3.3 实时K线订阅

- 对 `is_active=true` 的股票，建立富途实时K线推送订阅
- 推送回调写入当日最新 bar（`INSERT OR REPLACE`）
- 维护订阅状态表，防止超出账户额度上限（默认 100 只）
- 停用股票时同步取消订阅

### 3.4 复权处理

- **存储**：原始未复权价格，独立 `adjust_factors` 表存复权因子
- **计算**：`AdjustmentService.get_adjusted_klines()` 动态计算前复权价格序列
- **更新**：每次同步前刷新复权因子，仅追加新除权事件，不修改历史数据
- **原因**：避免每次除权后全量重刷历史 OHLCV，与 Wind/聚宽/掘金等业界主流一致

### 3.5 数据质量保障

- **校验**：OHLCV 合理性校验（high≥low、open/close 在 high-low 范围内、价格范围、成交量范围）
- **无效标记**：校验失败的 bar 标记 `is_valid=0`，不直接丢弃
- **空洞检测**：基于交易日历，检测并持久化数据缺口，支持重试修复
- **幂等写入**：历史拉取用 `INSERT OR IGNORE`，实时推送用 `INSERT OR REPLACE`

## 4. 非功能需求

| 项目 | 要求 |
|------|------|
| 数据库 | SQLite，WAL 模式，支持并发读 |
| 日志 | 按日轮转，保留 30 天，同时输出控制台 |
| 容错 | 限频/超时错误指数退避重试（1s/2s/4s），其他异常直接上报 |
| 可观测 | `sync_metadata` 表记录每次同步状态、行数、错误信息 |
| 环境隔离 | mamba 虚拟环境（`env_quant/`），不影响系统 Python |
| 敏感配置 | 通过 `.env` 文件注入，不提交仓库 |

---

## 5. 算法支持矩阵

| 算法类型 | 所需数据 | 满足情况 |
|---------|---------|---------|
| 技术分析（MA/MACD/RSI/KDJ/布林带）| OHLCV | ✅ 满足 |
| 量价因子（成交量异动、换手率）| volume, turnover, turnover_rate | ✅ 满足 |
| 动量/反转因子 | close, last_close | ✅ 满足 |
| 估值因子（PE）| pe_ratio（仅日K）| ✅ 满足（仅PE，无PB）|
| 基本面因子（ROE/营收增速）| 财务数据 | ❌ 不在本期范围 |
| 复权后价格序列 | 原始价格 + 复权因子 | ✅ 动态转换 |

---

## 6. 验收标准

1. `python -c "from db.schema import init_db; init_db('data/quant.db')"` 无报错，8张表正常创建
2. 首次运行 `python main.py`，日志无 ERROR，`kline_data` 有数据写入
3. `adjust_factors` 表有复权事件记录
4. `AdjustmentService.get_adjusted_klines()` 返回的前复权价格与富途 App 一致
5. 手动 DELETE 几天 `kline_data`，重跑后 `data_gaps` 记录被自动填补
6. 将某股票 `is_active` 改为 `false` 再改回 `true`，中间缺失日期被补齐
7. `subscription_status` 表确认活跃股票已订阅，日志出现推送回调记录
