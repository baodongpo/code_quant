### 2.4 FEAT-check-gaps-log：check-gaps 日志日期展示增强

#### 前置条件

- 虚拟环境 `env_quant/` 已激活或使用显式路径
- 数据库中有已知的历史 K 线数据（含 1W/1M 数据）
- 可访问日志文件 `logs/check_gaps_YYYYMMDD.log`

#### 测试数据准备说明

由于 check-gaps 依赖实际数据库空洞数据，建议以下两种方式之一：

**方式A（推荐，使用已有空洞数据）**：
- 若数据库 `data_gaps` 表中已有历史记录（status=open 或 closed），直接使用 `check-gaps` 命令扫描
- 若无，可先在测试数据库手动插入已知空洞：
  ```sql
  INSERT INTO data_gaps (stock_code, period, start_date, end_date, status)
  VALUES ('HK.00700', '1W', '2020-02-07', '2020-02-07', 'open');
  INSERT INTO data_gaps (stock_code, period, start_date, end_date, status)
  VALUES ('HK.00700', '1M', '2020-02-03', '2020-02-03', '1M', 'open');
  ```

**方式B（仅验证 _format_gap_date 逻辑）**：
- 添加单元测试或在 Python REPL 中直接验证辅助函数逻辑（无需真实数据库）

## 五、风险说明

### 5.1 需要人工肉眼判断的 AC

| 测试用例 | 判断内容 | 风险点 |
|---------|---------|--------|
| TC-align-02 | 十字线纵轴对齐肉眼判断 | 2px 以内的偏差难以肉眼区分，建议配合 TC-align-01 的 DevTools 像素测量 |
| TC-vpa-color-05 | 两种绿色视觉可区分性 | 主观判断，需 PM 或老板参与验收；在暗色背景下 #26a69a 与 #2ea043 差异较小 |
| TC-guide-icon-* | 图标形状（实线/虚线/圆/方块）目测区分 | 小图标在低分辨率下可能难以区分实线/虚线；建议使用 100% 缩放比例 |
| TC-reg-crosshair-* | 跨图十字线对齐 | 对齐是否"无明显错位"由人工判断，属主观标准 |

### 5.2 可辅助自动化验证的 AC

| 测试用例 | 验证方法 |
|---------|---------|
| TC-emoji-04 | 代码搜索（grep `\u2228` / `\u2227`），可写脚本自动检查 |
| TC-align-01 | 浏览器 Console 一键脚本获取所有 canvas 左边界坐标，计算差值 |
| TC-gaps-log-07 | Python 单元测试验证 `_format_gap_date` 函数逻辑 |
| TC-guide-7 | DevTools Console 检查有无 PropTypes 报错（可脚本化为 Playwright/Puppeteer 测试） |

### 5.3 功能点依赖风险

| 风险 | 描述 | 缓解措施 |
|------|------|---------|
| FEAT-guide-icon 依赖 FEAT-guide-top | 在 MACD/RSI/KDJ 面板中，图标验证需先有说明浮层（FEAT-guide-top 完成后才有） | 严格按开发顺序，FEAT-guide-top 合并后再测试 FEAT-guide-icon |
| FEAT-collapse-btn 与 FEAT-guide-top 共同修改面板标题行 | 若并行开发，代码合并时标题行 JSX 可能冲突 | 验收时若出现展开状态标题行布局异常，需排查是否为合并冲突遗留问题 |

### 5.4 BUG-align 特殊风险

- **高价股 Y 轴标签溢出**：A 股高价股（如贵州茅台，价格 > 1000 元）Y 轴标签可能超过 52px 宽度，被 `overflow: 'truncate'` 截断。测试时建议包含至少一只 A 股高价股验证 TC-align-03。
- **MACD 小数位数**：MACD DIF/DEA 值可能为 4 位小数（如 `0.0231`），约 7 字符，在 60px 内约为 49px 可正常显示，但极端情况下（如 `-0.0001`）可能超出。需测试空头行情下的 MACD 图。
