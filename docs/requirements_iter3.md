## 3. 技术指标清单

### 3.1 趋势类指标

---

#### T1. 移动平均线 MA（Moving Average）

**计算逻辑**：对收盘价取简单算术均值。`MA(n) = sum(close[i], i=0..n-1) / n`

**行业使用场景**：
- 判断趋势方向（价格在 MA 上方为多头）
- 金叉（短期上穿长期）/ 死叉（短期下穿长期）作为买卖信号
- 支撑/阻力位判断

**参数**：周期 n，常用组合：5、10、20、60、120、250（日K）

---

#### T2. 指数移动平均线 EMA（Exponential Moving Average）

**计算逻辑**：赋予近期价格更高权重的加权均线。`EMA(t) = close(t) × k + EMA(t-1) × (1-k)`，其中 `k = 2/(n+1)`

**行业使用场景**：
- 比 MA 对价格变化反应更灵敏，减少滞后
- MACD 的基础计算组件
- 短线趋势跟踪

**参数**：周期 n，常用：12、26

---

#### T3. MACD（Moving Average Convergence Divergence）

**计算逻辑**：
- `DIF = EMA(12) - EMA(26)`（快慢线差值）
- `DEA = EMA(DIF, 9)`（信号线）
- `MACD 柱 = (DIF - DEA) × 2`

**行业使用场景**：
- DIF 上穿 DEA（金叉）为买入信号，下穿（死叉）为卖出信号
- MACD 柱由负转正为趋势转换信号
- 顶背离/底背离是经典反转信号（价格创新高但 MACD 未创新高）
- A 股最广泛使用的趋势跟踪指标之一

**参数**：快线周期 12，慢线周期 26，信号线周期 9（业界标准，不建议修改）

---

#### T4. 布林带 BOLL（Bollinger Bands）

**计算逻辑**：
- 中轨 = `MA(20)`
- 上轨 = 中轨 + `2 × STD(close, 20)`
- 下轨 = 中轨 - `2 × STD(close, 20)`

**行业使用场景**：
- 价格触及上轨：超买区域，可能回调
- 价格触及下轨：超卖区域，可能反弹
- 带宽收窄（布林带收口）预示大幅波动即将来临
- 布林带突破（带宽扩大）确认新趋势启动

**参数**：均线周期 N=20，标准差倍数 K=2（最常用）

---

#### T5. 均线多空排列（MA Alignment）

**计算逻辑**：计算 MA5、MA10、MA20、MA60 的多头/空头排列状态。多头排列：`MA5 > MA10 > MA20 > MA60`

**行业使用场景**：
- 均线多头排列是上升趋势的强确认信号
- 均线空头排列是下降趋势的强确认信号
- A 股趋势判断必备辅助指标

**参数**：固定四条均线 5/10/20/60

---

### 3.2 震荡类指标

---

#### O1. RSI（Relative Strength Index，相对强弱指数）

**计算逻辑**：
- `RS = 平均上涨幅度 / 平均下跌幅度`（n日内）
- `RSI = 100 - 100 / (1 + RS)`

**行业使用场景**：
- RSI > 70：超买，价格可能回调（强势股可延伸到 80+）
- RSI < 30：超卖，价格可能反弹（弱势股可延伸到 20-）
- RSI 顶背离/底背离是中线反转信号
- 结合 MACD 使用，双重确认信号更可靠

**参数**：周期 n=14（最常用），可选 6、24

---

#### O2. KDJ（随机指标）

**计算逻辑**：
- `RSV(n) = (close - lowest_low(n)) / (highest_high(n) - lowest_low(n)) × 100`
- `K = 2/3 × K(prev) + 1/3 × RSV`
- `D = 2/3 × D(prev) + 1/3 × K`
- `J = 3K - 2D`

**行业使用场景**：
- K 线上穿 D 线（金叉，通常在 20 以下）为买入信号
- K 线下穿 D 线（死叉，通常在 80 以上）为卖出信号
- J 值超过 100 或低于 0 代表极端超买/超卖
- A 股短线操作最常用指标之一

**参数**：RSV 周期 n=9，K/D 平滑系数=3（业界标准）

---

#### O3. CCI（Commodity Channel Index，顺势指标）

**计算逻辑**：
- `典型价 TP = (high + low + close) / 3`
- `CCI = (TP - MA(TP, n)) / (0.015 × 平均绝对偏差)`

**行业使用场景**：
- CCI > +100：超买，可能短期回调
- CCI < -100：超卖，可能短期反弹
- CCI 从超买区域回落下穿 +100 为卖出信号
- 识别周期性的价格极端区域，港股/A 股均适用

**参数**：周期 n=20

---

### 3.3 成交量类指标

---

#### V1. OBV（On Balance Volume，能量潮）

**计算逻辑**：
- 若今日收盘 > 昨日收盘：`OBV = OBV(prev) + volume`
- 若今日收盘 < 昨日收盘：`OBV = OBV(prev) - volume`
- 若相等：`OBV = OBV(prev)`

**行业使用场景**：
- OBV 持续上升且价格上涨：量价齐升，趋势健康
- OBV 下降但价格上涨：量价背离，上涨不可持续（顶背离）
- OBV 领先价格变化，是价格趋势的先行指标
- 判断主力资金是否持续流入

**参数**：无（仅依赖 close 和 volume）

---

#### V2. 成交量均线 MAVOL

**计算逻辑**：对 volume 取移动平均。`MAVOL(n) = MA(volume, n)`

**行业使用场景**：
- 放量突破（成交量 > MAVOL × 1.5）是有效突破的确认信号
- 缩量回调后放量反弹为买入时机
- 量能萎缩判断横盘整理阶段

**参数**：周期 n，常用 5、10、20

---

### 3.4 波动率/通道类指标

---

#### W1. ATR（Average True Range，平均真实波幅）

**计算逻辑**：
- `真实波幅 TR = max(high-low, |high-close_prev|, |low-close_prev|)`
- `ATR(n) = MA(TR, n)`（或 EMA 版本）

**行业使用场景**：
- 衡量股票波动率，ATR 越大表示波动越剧烈
- 设置止损位：`止损 = 入场价 - K × ATR`（K=1.5~3）
- 判断是否处于突破还是震荡行情
- 量化策略中的头寸规模计算基础

**参数**：周期 n=14

---

#### W2. 布林带宽度 BBW（Bollinger Band Width）

**计算逻辑**：`BBW = (上轨 - 下轨) / 中轨 × 100`（基于 BOLL 计算）

**行业使用场景**：
- BBW 处于历史低位（布林带收口）：震荡整理末期，即将方向性突破
- BBW 快速扩大：趋势启动确认
- 结合价格方向判断突破方向

**参数**：依赖 BOLL 参数（N=20，K=2）

---

### 3.5 估值类指标（基于已有字段）

---

#### E1. PE/PB/PS 走势

**计算逻辑**：直接读取 `kline_data` 中已存储的 `pe_ratio`、`pb_ratio`（迭代2新增）、`ps_ratio`（迭代2新增）字段

**行业使用场景**：
- PE 历史百分位：当前 PE 处于历史第几百分位，判断相对高低估
- PB 结合 ROE（迭代4）：`PB/ROE` 是价值因子核心
- PS 走势判断成长股估值是否过高
- 与同行业平均值比较（迭代4+）

**参数**：历史回溯区间（如近 5 年）

| KDJ | 🟢金叉超卖 / 🔴死叉超买 / ⚖️中性 | 同个股分析页信号规范 |
| 综合信号 | 🟢偏多 / 🔴偏空 / ⚖️中性 | 三指标多数一致时显示对应信号 |

#### 综合信号判断逻辑（PM 定义，Dev 实现）

| 综合信号 | 判断条件 |
|---------|---------|
| 🟢 偏多 | MACD 金叉 且 RSI 在 50~70 中性偏多区间 且 KDJ 非死叉超买 |
| 🔴 偏空 | MACD 死叉 且 RSI < 50 且 KDJ 非金叉超卖 |
| ⚖️ 中性 | 其余情况（信号分歧或均处于中性区间）|

> **免责声明**：综合信号为技术指标的机械汇总，仅供参考，不构成投资建议。

## 5. 数据流说明

```
用户浏览器
    │ HTTP GET /api/kline?code=SH.600519&period=1D&start=...&adj=qfq
    ↓
后端 API 服务（新增，技术选型由 Dev 决定）
    ├─ 路由层：参数校验（stock_code 格式、period 枚举、日期格式）
    ├─ 服务层：调用 AdjustmentService.get_adjusted_klines()
    │              ↓ 读 kline_data 表（原始价格）
    │              ↓ 读 adjust_factors 表（复权因子）
    │              ↓ 动态计算前复权价格序列
    │         调用 IndicatorEngine.calculate(bars, indicators)
    │              ↓ 纯 Python 计算，不读库
    │              ↓ 返回各指标时间序列
    └─ 序列化为 JSON 返回
    │
    ↓ JSON 响应
前端（React）
    ├─ ECharts / Recharts 渲染 K 线图
    ├─ 叠加 BOLL / MA 等主图指标
    └─ 子图渲染 MACD / RSI / KDJ 等副图指标（含区间背景色、信号标识）
```

## 6. 技术要求（非功能性）

技术选型由 Dev 自主决定，须保证系统健壮性与可维护性。以下为产品层面的技术约束：

| 约束项 | 要求 |
|--------|------|
| 访问方式 | 浏览器通过 `http://localhost:{port}` 访问，端口号可通过 `.env` 配置 |
| 前后端 | 前后端分离架构，后端提供 REST API，前端负责渲染 |
| 数据库访问 | 后端直接读取现有 SQLite 数据库（只读），复用迭代1/2的 Repository 层 |
| 指标计算 | 在后端内存中计算，不写入数据库 |
| 前端图表 | 须支持 K 线蜡烛图、多子图联动、鼠标十字线，使用业界成熟图表库 |
| 跨域 | 前后端分离开发时须处理 CORS |
| 生产部署 | 后端可直接 serve 前端构建产物（静态文件），单进程运行 |

## 7. 非功能需求

| 项目 | 要求 | 说明 |
|------|------|------|
| 响应时间 | 单股单指标 < 1秒 | K线数据本地 SQLite 读取，指标纯内存计算，无网络依赖 |
| 浏览器兼容 | Chrome / Safari / Edge 最新版 | 个人工具，不需要 IE/旧浏览器支持 |
| 并发 | 单用户（个人工具）| 无需高并发设计 |
| 数据安全 | 仅 localhost 访问 | 不暴露外网，无需认证鉴权 |
| 移动端 | 不要求 | 桌面浏览器优先 |
| 数据库只读 | 严格只读 | Web 服务不写入任何数据，数据库由迭代1/2 的采集服务独立维护 |
| 稳定性 | 与数据采集服务并行运行 24 小时不崩溃 | 两个服务进程独立，互不影响 |

## 8. 不做什么（范围边界）

| 不做 | 原因 |
|------|------|
| 自动交易、下单 | 系统核心约束，严禁 |
| 用户认证/多用户 | 个人工具，localhost 访问 |
| 告警推送（微信/邮件）| 迭代4+ 规划 |
| 基本面数据展示 | 迭代4+ 与 ROE 数据一起规划 |
| 策略回测 | 超出数据源子系统定位，属于独立系统 |
| 实时 WebSocket 推送 | 60秒轮询已足够日K级别使用场景 |
| 自定义指标公式编辑器 | 复杂度高，非核心需求 |
| 数据导入/修改功能 | Web 层严格只读 |
