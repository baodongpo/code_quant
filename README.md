# AI 量化辅助决策系统 — 数据源子系统

> **声明**：本系统仅负责数据采集与存储，严禁包含任何自动下单、报价、交易相关逻辑。

---

## 这是什么

一套面向量化策略研究的**本地 K 线数据服务**，以富途 OpenD 为数据源，将 A股、港股、美股的历史与实时 K 线数据落库到本地 SQLite，并提供动态前复权能力，供下游算法直接消费。

**解决的核心问题**：

- 量化策略需要干净、连续、经过复权处理的历史价格序列，但手工维护成本高
- 除权事件频繁，复权因子的管理和历史价格的一致性难以保证
- 多市场（A股 / 港股 / 美股）交易日历不同，数据空洞难以系统检测

## 已实现功能说明

### Watchlist 管理

编辑根目录 `watchlist.json` 维护关注列表，每次运行 `main.py` 自动检测变化：

- **新增股票**：触发全量历史拉取（从 2015-01-01 起）
- **重新激活**（`is_active: false → true`）：从首次同步日期开始检测并补全空洞
- **停用**（`is_active: true → false`）：暂停同步，保留已有历史数据

### 历史 K 线采集

- 粒度：日 K（1D）、周 K（1W）、月 K（1M）
- 字段：开盘、最高、最低、收盘、成交量（股）、成交额、换手率、PE（TTM，仅日K）、前收盘价
- 分页自动处理，单次最多 1000 条

### 复权处理

采用业界主流的**存储原始价格 + 独立复权因子表**策略，与 Wind/聚宽/掘金一致：

```
前复权价格(t) = 原始价格(t) × ∏{ forward_factor(i) | ex_date(i) > t }
```

优势：除权后只需插入一条复权因子记录，历史 OHLCV 数据无需重刷。

算法层调用方式：

```python
AdjustmentService.get_adjusted_klines(stock, period, start, end, adj_type="qfq")
# 返回前复权后的 KlineBar 列表，不写库
```

### 空洞检测

以交易日历（非自然日历）为基准，检测 `kline_data` 中缺失的交易日，持久化到 `data_gaps` 表，下次同步自动修复。

### 数据质量校验

每条 K 线写入前进行以下校验，失败标记 `is_valid=0` 而非丢弃：

- high ≥ low，open/close 在 high-low 范围内
- 价格、成交量在合理区间

## 数据库结构

共 7 张表：

| 表名 | 说明 |
|------|------|
| `stocks` | 股票基础信息（代码、名称、市场、lot_size 等）|
| `kline_data` | K 线原始数据（未复权）|
| `adjust_factors` | 复权因子（每次除权事件一条记录）|
| `trading_calendar` | 各市场交易日历 |
| `sync_metadata` | 每只股票每个周期的同步状态与进度 |
| `data_gaps` | 检测到的数据空洞记录 |
| `subscription_status` | 实时订阅状态（迭代 2 启用）|

## 快速开始

### 1. 启动 OpenD

打开富途牛牛客户端，确保 OpenD 正在运行（默认监听 `127.0.0.1:11111`）。

### 2. 创建虚拟环境

```bash
mamba create -p ./env_quant python=3.10 -y
mamba activate ./env_quant
pip install -r requirements.txt
```

### 3. 初始化配置

```bash
# 环境配置
cp .env.example .env
# 按需修改 OPEND_HOST / OPEND_PORT 等

# Watchlist 配置
cp watchlist.json.example watchlist.json
# 编辑 watchlist.json，填入实际关注的股票
```

### 4. 运行

```bash
python main.py
```

### 5. 查看日志

```bash
tail -f logs/sync_$(date +%Y%m%d).log
```

## 配置说明

所有配置通过根目录 `.env` 文件注入，参考 `.env.example`：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `OPEND_HOST` | `127.0.0.1` | OpenD 监听地址 |
| `OPEND_PORT` | `11111` | OpenD 监听端口 |
| `DB_PATH` | `data/quant.db` | SQLite 数据库路径 |
| `DEFAULT_HISTORY_START` | `2015-01-01` | 历史拉取起始日 |
| `MAX_SUBSCRIPTIONS` | `100` | 最大实时订阅数 |
| `RATE_LIMIT_MIN_INTERVAL` | `0.5` | 请求最小间隔（秒）|
| `RATE_LIMIT_MAX_IN_WINDOW` | `25` | 30秒窗口内最大请求数 |
